PHASE 1 : INSTALLATION
================================================================================

ÉTAPE 1.1 - Cloner le projet
─────────────────────────────
Commande:
  git clone https://github.com/votre-repo/logmonitor.git
  cd logmonitor

Résultat attendu:
  - Dossier logmonitor créé avec tous les fichiers du projet


ÉTAPE 1.2 - Créer l'environnement virtuel Python
──────────────────────────────────────────────────
Commande:
  python -m venv venv

Activation Windows:
  venv\Scripts\activate

Activation Linux/macOS:
  source venv/bin/activate

Résultat attendu:
  - L'invite de commande affiche le préfixe (venv)
  - L'environnement virtuel est isolé


ÉTAPE 1.3 - Installer les dépendances Python
──────────────────────────────────────────────
Commande:
  pip install -r requirements.txt

Résultat attendu:
  - Tous les packages listés dans requirements.txt sont installés
  - Aucune erreur "ERROR: Could not find a version"


ÉTAPE 1.4 - Configurer les variables d'environnement
──────────────────────────────────────────────────────
Commande:
  cp .env.example .env

Éditer le fichier .env avec les paramètres minimum:
  SECRET_KEY=votre-secret-key-ici
  DEBUG=True
  DATABASE_URL=sqlite:///db.sqlite3

Pour générer une SECRET_KEY sécurisée:
  python -c "import secrets; print(secrets.token_hex(50))"

Résultat attendu:
  - Fichier .env créé dans la racine du projet
  - Variables d'environnement correctement configurées


ÉTAPE 1.5 - Initialiser la base de données
────────────────────────────────────────────
Commande:
  python manage.py migrate

Résultat attendu:
  - Migrations appliquées avec succès (dont accounts.0002_alter_user_role)
  - Fichier db.sqlite3 créé (si SQLite)
  - Tables créées dans PostgreSQL (si utilisé)
  - Pas d'erreurs de migration


ÉTAPE 1.6 - Charger les données de démonstration

Commande:

Données créées:

  ✓ 1000 logs réalistes sur 48 heures
  ✓ 2 alertes actives
  ✓ Compte admin par défaut: admin / password123

Résultat attendu:
  - Commande exécutée sans erreurs
  - Données insérées dans la base de données
  - Compte admin accessible

================================================================================
NOTES IMPORTANTES
================================================================================

1. SECRET_KEY
   - Générer une nouvelle clé sécurisée pour la production
   - Ne jamais commiter la vraie clé

2. DEBUG
   - DEBUG=True en développement uniquement
   - DEBUG=False en production

3. DATABASE_URL
   - sqlite:///db.sqlite3 pour développement
   - postgresql://user:pwd@host:5432/db pour production

4. Sécurité CSRF
   - Activée par défaut sur tous les formulaires
   - Logout POST-only (protection contre CSRF logout)

5. Auto-refresh HTMX
   - Tableau des logs mis à jour toutes les 30s
   - Peut être désactivé côté client si nécessaire

6. Rôles utilisateurs
   - L'inscription publique attribue automatiquement le rôle "Utilisateur"
   - Seul un Administrateur peut changer le rôle d'un compte
   - Le compte admin par défaut (admin/password123) est protégé :
     son rôle ne peut pas être modifié et il ne peut pas être supprimé

7. Logs de démonstration
   - 1000 logs couvrent 48 heures
   - Timestamp généré aléatoirement dans cette période
   - Peut être augmenté avec: seed_data --logs 5000

8. Support navigateurs
   - Chrome/Edge 80+
   - Firefox 75+
   - Safari 13+
